## 一、`pom.xml`：依赖管理（项目的"购物清单"）

### 🛒 什么是 pom.xml？

> 你可以把 `pom.xml` 想象成**项目的购物清单**，告诉 Maven（包管理工具）：「请帮我下载这些第三方库」。每加一个 `<dependency>`，就相当于在清单上多写一行商品。

### 改动前 vs 改动后

**🔴 删除了**：
```xml
<dependency>
    <groupId>org.mybatis.spring.boot</groupId>
    <artifactId>mybatis-spring-boot-starter</artifactId>
    <version>4.0.1</version>
</dependency>
```
> 删掉的是**原生 MyBatis**，因为我们要用更强大的「MyBatis-Plus」（在 MyBatis 基础上做了增强，比如自动生成 CRUD 方法）。

**🟢 新增了 3 个依赖**：

#### ① MyBatis-Plus（数据库操作工具）

```xml
<dependency>
    <groupId>com.baomidou</groupId>
    <artifactId>mybatis-plus-spring-boot4-starter</artifactId>
    <version>3.5.16</version>
</dependency>
```

**作用**：让 Java 代码方便地操作数据库（增删改查）。

**为什么要用 Plus 版本？**
- 原生 MyBatis：每个 SQL 都要手写
- MyBatis-Plus：内置了常用的增删改查方法，少写大量重复代码

**⚠️ 注意点**：`spring-boot4-starter` 中的 `4` 表示这个版本是给 **Spring Boot 4.x** 用的，不要用错版本（Spring Boot 3 要用 `spring-boot3-starter`）。

#### ② Knife4j（API 文档页面）

```xml
<dependency>
    <groupId>com.github.xiaoymin</groupId>
    <artifactId>knife4j-openapi3-jakarta-spring-boot-starter</artifactId>
    <version>4.5.0</version>
    <exclusions>
        <exclusion>
            <groupId>org.springdoc</groupId>
            <artifactId>springdoc-openapi-starter-webmvc-ui</artifactId>
        </exclusion>
    </exclusions>
</dependency>
```

**作用**：自动给你的项目生成一个**漂亮的 API 文档网页**，访问 `http://localhost:8080/doc.html` 就能看到所有接口。

**🤔 那个 `<exclusions>` 是干嘛的？**

> 把 Knife4j 想象成「**手机壳**」，它需要套在「**手机（SpringDoc）**」上。
> Knife4j 4.5.0 自带的"手机"是 SpringDoc 2.3.0，但这个旧手机和我们的新系统（Spring Boot 4）不兼容。
> 所以我们用 `<exclusions>` 告诉 Maven：**"不要装它自带的旧手机"**，下面我们手动装新的。

#### ③ SpringDoc 3.0.0（手动装的"新手机"）

```xml
<dependency>
    <groupId>org.springdoc</groupId>
    <artifactId>springdoc-openapi-starter-webmvc-ui</artifactId>
    <version>3.0.0</version>
</dependency>
```

**作用**：扫描你的 Controller，把接口信息转换成 OpenAPI 标准格式，再交给 Knife4j 渲染。

**为什么用 3.0.0？**：这是**唯一适配 Spring Boot 4** 的版本（2025 年 11 月才发布）。

## 二、`application.properties`：项目配置（"使用说明书"）

### 🔧 什么是 application.properties？

> Spring Boot 启动时会读这个文件，按里面的规则做事。比如 `server.port=8080` 就是告诉 Spring Boot：「请把服务跑在 8080 端口」。

### 改动详解（按区块分组）

#### ① 服务端口

```properties
server.port=8080
```

> 项目启动后监听 8080 端口。访问地址 `http://localhost:8080`。

#### ② MySQL 数据源（怎么连数据库）

```properties
spring.datasource.url=jdbc:mysql://localhost:3306/light-blog?useSSL=false&serverTimezone=Asia/Shanghai&allowPublicKeyRetrieval=true
spring.datasource.username=root
spring.datasource.password=1234qwer
spring.datasource.driver-class-name=com.mysql.cj.jdbc.Driver
```

| 配置项 | 含义 | 类比 |
|-------|------|-----|
| `url` | 数据库地址 | 你家的门牌号 |
| `username` | 用户名 | 你家的钥匙 |
| `password` | 密码 | 钥匙的密码 |
| `driver-class-name` | 驱动类 | 用什么车去你家 |

**⚠️ URL 后面那一串 `?useSSL=...` 是啥？**
- `useSSL=false`：不要求 SSL 加密（本地开发用）
- `serverTimezone=Asia/Shanghai`：时区设为上海，避免时间差 8 小时
- `allowPublicKeyRetrieval=true`：MySQL 8 必须加这个，否则连不上

#### ③ MyBatis-Plus 配置

```properties
mybatis-plus.mapper-locations=classpath*:/mapper/**/*.xml
mybatis-plus.type-aliases-package=com.example.demo.entity
mybatis-plus.configuration.map-underscore-to-camel-case=true
mybatis-plus.configuration.log-impl=org.apache.ibatis.logging.stdout.StdOutImpl
```

| 配置项 | 含义 |
|-------|------|
| `mapper-locations` | SQL 模板文件（XML）放在哪里 |
| `type-aliases-package` | 实体类（数据库表对应的 Java 类）放在哪个包 |
| `map-underscore-to-camel-case` | 数据库字段 `user_name` ↔ Java 字段 `userName` 自动转换 |
| `log-impl` | 把执行的 SQL 打印到控制台（方便调试） |

#### ④ Knife4j / SpringDoc 配置

```properties
springdoc.api-docs.enabled=true
springdoc.api-docs.path=/v3/api-docs
springdoc.swagger-ui.enabled=true
springdoc.swagger-ui.path=/swagger-ui.html
springdoc.packages-to-scan=com.example.demo
knife4j.enable=true
knife4j.setting.language=zh_cn
```

| 配置项 | 含义 |
|-------|------|
| `api-docs.path` | OpenAPI JSON 数据的访问路径 |
| `swagger-ui.path` | 原生 Swagger UI 的访问路径 |
| `packages-to-scan` | 扫描哪个包下的 Controller 来生成文档 |
| `knife4j.enable` | 启用 Knife4j 增强 UI |
| `setting.language` | 文档界面用中文 |

**📍 三个文档地址分别是什么？**

| URL | 是什么 |
|-----|--------|
| `http://localhost:8080/doc.html` | **Knife4j 美化版**（推荐用这个） |
| `http://localhost:8080/swagger-ui.html` | 原生 Swagger UI（基础版） |
| `http://localhost:8080/v3/api-docs` | 给程序看的 JSON 数据（不是给人看的） |

## 三、`DemoApplication.java`：主启动类

### 改动内容

```diff
  package com.example.demo;

+ import org.mybatis.spring.annotation.MapperScan;
  import org.springframework.boot.SpringApplication;
  import org.springframework.boot.autoconfigure.SpringBootApplication;

  @SpringBootApplication
+ @MapperScan("com.example.demo.mapper")
  public class DemoApplication {

      public static void main(String[] args) {
          SpringApplication.run(DemoApplication.class, args);
+         System.out.println("Hello World!");
      }
  }
```

### 为什么要加 `@MapperScan`？

> **MyBatis 的 Mapper 接口很特殊**：它只是一个 Java interface，没有实现类，但 MyBatis 会在运行时**自动给它生成实现**。
>
> Spring 不知道这种"魔法"，所以你要告诉它：
> **「`com.example.demo.mapper` 包里的所有接口，都交给 MyBatis 处理」**

🌰 举个例子，以后你创建一个：
```java
package com.example.demo.mapper;

public interface UserMapper extends BaseMapper<User> {
    // 这里什么都不用写！
}
```
有了 `@MapperScan` 后，Spring 就会自动帮你 new 一个实例出来给你用。

### `System.out.println("Hello World!")` 是干嘛的？

> 这行是你后来手动加的，作用是项目启动后在控制台打印一句"Hello World!"，可以验证启动是否成功。生产环境一般会删掉。

## 四、过程中踩的两个坑（重点理解！）

### 🕳️ 坑 1：MyBatis-Plus 一开始用了 3.5.12，IDE 报红

**现象**：`Dependency 'mybatis-plus-spring-boot4-starter:3.5.12' not found`

**原因**：阿里内部 Maven 仓库里没有 3.5.12 这个版本。

**解决**：换成阿里仓库里有的 **3.5.16**。

**📚 经验**：
- 选依赖版本时，**先去 [mvnrepository.com](https://mvnrepository.com) 看看哪些版本被广泛使用**
- IDE 报红不一定是依赖错了，也可能是 IDE 索引没刷新（**右键 pom.xml → Maven → Reload Project**）

---

### 🕳️ 坑 2：访问 `/doc.html` 出现 Whitelabel Error Page（最大的坑！）

**现象**：项目能启动，但 `http://localhost:8080/doc.html` 返回 404。

**原因（重点理解）**：

```
我们用了 Spring Boot 4.0.6 → 它内部使用 Jackson 3（最新的 JSON 库）
        ↓
Knife4j 4.5.0 自动带来的 SpringDoc 2.3.0 → 它使用的是旧的 Jackson 2
        ↓
两者打架 → SpringDoc 启动失败 → /doc.html 没有被注册 → 404
```

> 💡 **类比**：就像你买了一台 USB-C 接口的电脑（Spring Boot 4），但配的鼠标是 USB-A 接口的（旧 SpringDoc），插不上就用不了。

**解决步骤**：

1. **拆掉旧鼠标**：用 `<exclusions>` 排除 Knife4j 自带的 SpringDoc 2.3.0
2. **装上新鼠标**：手动引入 SpringDoc 3.0.0（这个版本支持 Spring Boot 4）

**📚 经验**：
- 用 `./mvnw dependency:tree` 命令可以看到**所有依赖关系**，能直观发现冲突
- 遇到"莫名其妙"的 404，**先怀疑依赖冲突**，再怀疑配置错误
- **大版本升级（如 Spring Boot 3 → 4）时，所有第三方库都要重新检查兼容性**

## 五、🎓 总结：本次学到了什么

### 知识点

1. **Maven 依赖管理**
   - `<dependency>` 添加依赖
   - `<exclusions>` 排除传递依赖（解决冲突的关键）
   - `dependency:tree` 命令排查依赖

2. **Spring Boot 配置**
   - `application.properties` 是项目配置中心
   - 命名规则：`框架名.模块.参数=值`

3. **MyBatis-Plus 集成三步曲**
   - 加依赖 → 配数据源 → 加 `@MapperScan`

4. **Knife4j 工作原理**
   - Knife4j = 美化的前端 UI
   - SpringDoc = 扫描 Controller 生成数据
   - 两者必须配套，版本要兼容

### 调试技巧

| 场景 | 命令 / 操作 |
|------|------------|
| 依赖看起来红色 | IDE 右键 pom.xml → Maven → Reload Project |
| 想看实际下载了哪些依赖 | `./mvnw dependency:tree` |
| 想确认能不能编译 | `./mvnw clean compile` |
| `/doc.html` 404 | 检查 SpringDoc 版本是否兼容 Spring Boot 版本 |
